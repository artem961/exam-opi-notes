# Билет 63. Тестирование системы в целом. Системное тестирование. Тестирование производительности

## Ответ

### Системное тестирование

**Системное тестирование** — тестирование полностью интегрированной системы на соответствие требованиям. Выполняется на системе целиком в окружении, близком к production.

Что проверяется:
- Функциональные требования (все функции работают).
- Нефункциональные требования (производительность, надёжность, безопасность, usability).
- Взаимодействие с внешними системами.
- Установка и конфигурация.

Уровень в V-модели: системное тестирование ↔ системное проектирование.

### Тестирование производительности

**Тестирование производительности** — проверка того, что система выдерживает заданные нагрузки и укладывается в допустимое время ответа.

#### Виды

| Вид | Цель |
|-----|------|
| **Нагрузочное (Load)** | Проверить систему при ожидаемой нагрузке (N пользователей) |
| **Стрессовое (Stress)** | Найти предел: при какой нагрузке система ломается |
| **Объёмное (Volume)** | Проверить с большим объёмом данных (миллион записей в БД) |
| **Нагрузочное пиковое (Spike)** | Резкий скачок нагрузки: выдержит ли система всплеск |
| **Длительное (Soak/Endurance)** | Продолжительная нагрузка: нет ли утечек памяти за 8 часов |

#### Ключевые метрики

```
Время ответа (Response Time) — 95-й перцентиль < 2 с
Пропускная способность (Throughput) — 1000 RPS
Количество ошибок (Error Rate) — < 1%
Использование ресурсов — CPU < 80%, RAM < 70%
```

#### Инструменты нагрузочного тестирования

- **JMeter** — десктопный, Java, поддерживает HTTP/HTTPS/JDBC/FTP.
- **Gatling** — Scala, высокая нагрузка, хорошие отчёты.
- **k6** — JavaScript, легковесный, удобен для разработчиков.
- **Apache Bench (ab)** — простой, встроен в Apache, для быстрых проверок.

## Подробно

### Что входит в системное тестирование

Помимо функциональных тестов системное тестирование включает:

**Тестирование безопасности:**
- SQL-инъекции, XSS, CSRF.
- Авторизация: нельзя получить чужие данные.
- Пентест с известными уязвимостями.

**Тестирование совместимости:**
- Разные браузеры (Chrome, Firefox, Safari, Edge).
- Разные ОС (Windows, macOS, Linux).
- Разные разрешения экрана.

**Тестирование установки:**
- Чистая установка работает.
- Обновление с предыдущей версии работает.
- Удаление оставляет систему в чистом состоянии.

### JMeter: структура теста

```
Test Plan
  └─ Thread Group (100 пользователей, 60 секунд)
       ├─ HTTP Request: GET /api/products
       ├─ HTTP Request: POST /api/cart
       └─ Listener: Summary Report (время, RPS, ошибки)
```

### Перцентили vs среднее

Среднее время ответа обманывает: если 95% запросов выполняются за 100 мс, а 5% — за 10 секунд, среднее будет ~600 мс, что скрывает проблему. Правильно смотреть:
- **P50** (медиана): 50% быстрее этого времени.
- **P95**: 95% запросов быстрее, 5% — нет. Обычно это «реальное» время для пользователя.
- **P99**: 99% быстрее. Показывает «хвост» распределения.

Требования обычно формулируют как: «P95 < 500 мс при 1000 RPS».

### Базовая линия (Baseline)

Перед нагрузочным тестированием устанавливают базовую линию: измеряют производительность текущей версии. После изменений сравнивают с базовой — это позволяет заметить деградацию.
