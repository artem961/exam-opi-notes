# Билет 1. Модели жизненного цикла ПО

## Ответ

Жизненный цикл ПО — период от замысла системы до вывода её из эксплуатации. Модель ЖЦ описывает, как организованы этапы разработки и как они связаны между собой.

Выделяют три основных класса моделей.

**Последовательная (Waterfall)** — этапы идут строго один за другим: анализ → проектирование → реализация → тестирование → сопровождение. Следующий этап начинается только после завершения предыдущего. Проста в управлении, но негибка: изменить требования после старта дорого.

**Инкрементная** — система строится и поставляется частями. Каждый инкремент добавляет новую функциональность к уже работающей версии. Заказчик получает результат раньше и может давать обратную связь по ходу.

**Эволюционная** — разработка идёт итерациями, каждая проходит через анализ, проектирование, реализацию и оценку. На каждом витке система уточняется. Подходит, когда требования нечёткие или меняются.
